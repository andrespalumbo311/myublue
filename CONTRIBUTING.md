# AI Contribution Guidelines

This repository has a rigid structure based on Fedora Atomic and containerized builds. Any AI agent interacting with this codebase **must** adhere to these rules to avoid breaking the build pipeline.

## Mandatory Operational Rules
1. **Verify Documentation:** Before taking any action, read this file and the "Historical Errors & Prevention" table.
2. **Self-Update:** If you commit an error, you **must** update the table in this file. If the error belongs to an existing class, refine the "Preventive Action" with the new technical insights. The table is a **living library of architectural patterns**, not a chronological log of instances.
3. **Technical Abstraction:** Errors must be documented in a **general and abstract form**. Describe the underlying technical logic (e.g., "Flatpak remote metadata caching") so the solution acts as a reusable pattern for any similar scenario in the project.
4. **Live Session Validation:** For any changes affecting graphical sessions, portals, or user-level services, the agent **must** perform live validation (e.g., using `busctl`, `systemctl --user`, or `gsettings`) to confirm the backend is active and exposing the correct interfaces *before* committing.

## Historical Errors & Prevention

| Error Class / Technical Challenge | Root Cause (Abstract) | Preventive Action / Architectural Pattern |
| :--- | :--- | :--- |
| **Vital Directory Persistence** | Git does not track empty directories; deleting the last file in a mapped directory breaks `COPY` instructions. | Always maintain a `.keep` file in directories that must persist in the repository structure. |
| **Package Source & Namespace Integrity** | Assuming package availability, naming, or external installer flags without verification. Third-party COPRs often diverge from standard Fedora naming conventions (e.g., missing `-modules-extra`). | 1. **Verify** exact package names via `dnf list` simulation or official repo search. 2. For GitHub releases, check actual asset names. 3. When switching from RPM to standalone binaries, **always** manually provide missing systemd units (`.service`, `.timer`) and config files previously included in the package. |
| **Application ID Mismatch** | Linking services using assumed IDs instead of verified Flatpak IDs causes MIME and Portal failures. | **Always** align symlinks and MIME integrations with the official ID. Use shadow `.desktop` files for internal app checks if needed. |
| **Flatpak Remote Persistence & Indexing** | In atomic systems, simply adding a remote configuration file may not trigger metadata indexing. | Use a hybrid approach: add the `.flatpakrepo` file to `etc/` **and** run `remote-add` + `update --appstream` during the build to warm the cache. |
| **Third-Party Repository Validation** | Relying on unverified URLs or corrupted GPG keys for external repositories causes build failures. | Always verify raw repository configuration files and check URL accessibility (e.g., via `curl -I`) before integration. |
| **Plugin Dependency Resolution** | Software plugins often depend on low-level CLI utilities that are not included in base images. | Audit plugin source code or documentation to identify and explicitly install all required system-level CLI dependencies in the build manifest. |
| **Configuration Include Integrity** | Modular configuration files using `include` statements will fail if any referenced path is missing. | Ensure all included configuration fragments exist in the repository, using empty placeholders if necessary. |
| **Atomic System GPG Keyring Constraints** | Initializing a remote via URL during build may fail if the GPG key is not yet trusted. | Point the initialization command to a local `.flatpakrepo` file that already contains the GPG key to ensure atomic import and trust. |
| **Topgrade NPM Symlink Conflict** | `topgrade` automatically detects and updates `npm` globally. If `npm` is a Homebrew dependency, it conflicts with Homebrew's symlinks in `share/man`. | Disable `npm` in `topgrade.toml` to let Homebrew manage its own dependencies exclusively. **Note**: When using a custom `topgrade.toml`, you MUST explicitly set `bootc = true` in the `[linux]` section for Atomic systems. |
| **Config Redirection Corruption** | Using append (`>>`) on configuration files without checking for existing sections causes "duplicate key" syntax errors. | **Always** verify file content before redirecting or use structured editing (e.g., `sed` or `yq`) to modify existing configuration blocks. |
| **Missing Remote Desktop Backend (wlroots)** | Remote control (Valent) fails on Niri if the portal does not export the `RemoteDesktop` interface. | **Use** `xdg-desktop-portal-hyprland` for full `RemoteDesktop` support on wlroots and ensure `/dev/uinput` permissions via `tmpfiles.d`. |
| **Immutable Path Conflict** | Attempting to create a directory where a symlink exists (or vice versa) on an atomic system causes failures. | Use `tmpfiles.d` for path redirection or ensure the build stage correctly handles the target distribution's symlink structure. |
| **Transition Package Naming** | Assuming package names based on previous versions (e.g., KF5 vs KF6) leads to non-existent package errors. | Always verify exact package names on the target host or official repositories, especially for toolkit-specific libraries. |
| **Build Environment Leakage** | Failing to isolate build caches (like `CARGO_HOME`) in multi-stage builds leads to permission errors or bloated layers. | Explicitly define and isolate build environment variables and cache directories within the `builder` stage. |
| **Inconsistent Release Assets (Verification Failure)** | Relying on external signature files (`.sig`) or checksum files (`.sha256`) causes 404 errors if projects change their release naming conventions or stop providing them. | **Use the GitHub API `digest` field** to programmatically retrieve and verify the SHA256 of any release asset. This provides a unified, version-agnostic, and robust verification method that doesn't depend on the presence of sidecar files. |
| **Stale Context for Surgical Edits** | Attempting surgical file edits (e.g., via `replace`) using context derived from a state prior to a file system operation (like `git restore` or `git checkout`) causes tool failure due to mismatched content. | **Re-read files** or verify their current state after any git command that modifies the working tree before attempting further automated code changes. |
| **Sandboxed App Discovery** | Placing `.desktop` files in non-standard paths prevents detection of sandboxed applications (Flatpaks). | Ensure all application entries are placed in or linked to standard XDG paths verified by the desktop environment. |
| **Duplicate Flatpak Icons** | Creating manual symlinks for Flatpak `.desktop` files in `XDG_DATA_DIRS` causes duplicate launcher entries. | **Do not** manually link Flatpak-exported desktop files; rely on standard export paths or use `flatpak override`. |
| **Persistent Home Overrides** | System-wide MIME associations in `/etc/skel` are ignored if the user has a pre-existing `~/.config/mimeapps.list`. | Use systemd setup services to explicitly patch existing user configuration files in `/var/home` if a default must be enforced. |
| **Shell Configuration Drift** | Placing critical CLI tool initializations in `/etc/skel` fails to update existing users during image upgrades. | Move system-wide shell initializations (interactive features, aliases) to `/etc/fish/conf.d/` or equivalent system paths to ensure all users receive updates. |
| **Locked Keyring on Wayland** | Compositors like Niri don't automatically unlock the GNOME Keyring. Incorrect XDG portal backend names (e.g. `gnome-keyring` instead of `gnome`) break Secret portal integration for Flatpaks. | 1. Configure PAM (`/etc/pam.d/greetd`). 2. Use `org.freedesktop.impl.portal.Secret=gnome` (not `gnome-keyring`). 3. Explicitly export `GNOME_KEYRING_CONTROL` to D-Bus/Systemd. |
| **SELinux Denials in greetd** | Custom PAM files for `greetd` without `pam_selinux.so` prevent the session from transitioning to the user context, blocking access to systemd units. | **Always** include `pam_selinux.so` (open/close) in custom PAM configs on Fedora-based systems to ensure correct context transition. |
| **Niri Spawn Path Failure** | Niri's `spawn-at-startup` does not expand `~`. Using relative or tilde paths causes the startup to fail silently or crash the session. | Use `spawn-sh-at-startup` with `$HOME` for scripts located in user directories to ensure proper shell expansion. |
| **Orphaned File References (Build Failure)** | Deleting or renaming a repository asset without updating all references in the build manifest (Containerfile, scripts) causes execution failures during the build process. | **Always** perform a global search (grep) for the filename or path of a deleted/renamed asset to identify and update all associated commands (e.g., `chmod`, `chown`, `COPY`). |
| **Missing Build Instruction (Syntax Error)** | Accidental removal of mandatory instructions (e.g., `RUN`, `COPY`) during surgical edits causes the build engine to interpret shell scripts as invalid keywords (e.g., `Error: Unknown instruction: IF`). | **Verify** that every functional block in the Containerfile is preceded by a valid instruction. Use `bootc container lint` locally or via simulation if possible after edits. |
| **Containerized GRUB Probing** | Post-install scripts of kernel/grub RPMs fail in containers as they cannot resolve the canonical path of the `overlay` filesystem. | **Mock** `grub2-probe` and `grub2-editenv` in a temporary directory (e.g., `/tmp/bin`) and prepend it to the **PATH** during the `dnf` transaction to override standard system binaries. |
| **Ephemeral Path Persistence** | Including files or directories in `/run`, `/tmp`, or `/var` directly in the image triggers `bootc lint` warnings and violates stateless principles. | **Never** persist ephemeral content in the image. Use `/usr/lib/tmpfiles.d/*.conf` to declare required directories or files, ensuring the system creates them dynamically at runtime. |
| **Monitor Alignment & Pointer Traps** | Defining monitor positions that only touch at a single logical point (e.g., corner-to-corner) or using auto-snapping features that disregard vertical layouts. | 1. Ensure monitor boundaries share a logical edge (not just a vertex). 2. Disable third-party "monitor snapping" features (e.g., in DMS) when using vertical or custom layouts to prevent unintended horizontal offsets. 3. Validate KDL syntax (use `x=0 y=0`) to prevent fallback to automatic horizontal positioning. |
| **SCX vs Ananicy-cpp Conflict** | Running both an eBPF scheduler (SCX) and a priority daemon (Ananicy) causes "priority fighting". Firefox/Browsers experience multi-second hangs during media initialization. | **Never** run Ananicy-cpp alongside SCX. Modern eBPF schedulers handle process priorities natively at the kernel level. |
| **TLP vs SCX/P-State Latency** | TLP's aggressive power saving on battery limits CPU scaling, which conflicts with SCX's high-frequency scheduling demands. This causes UI stuttering and browser lag. | Use **power-profiles-daemon** instead of TLP. It integrates natively with the kernel P-State driver and modern desktop environments/schedulers. |
| **GPU Driver Case Sensitivity** | `LIBVA_DRIVER_NAME` is case-sensitive. Using `ihd` instead of `iHD` for Intel Media Driver causes fallback to software rendering. | **Always** use `iHD` (exact case) for Intel Gen8+ hardware acceleration. Ensure the package is named **`libva-intel-media-driver`** in official Fedora repos. |
| **Shell Resource Instability** | Enabling real-time audio visualization in DankMaterialShell (DMS) causes ~60% CPU overhead and OSD ghost popups during IPC events. | **Disable** `audioVisualizerEnabled` and `soundVolumeChanged` triggers in `settings.json` to maintain shell stability. |
| **Greetd PAM Auth Failure** | Using `substack system-auth` for the `greetd` user (who typically has no password) prevents the greeter from launching. | Use `pam_permit.so` in `/etc/pam.d/greetd` to allow the greeter user to initialize the session without a password. |
| **Broken Tmpfiles Symlinks** | Declaring `L+` (link) in `tmpfiles.d` to a path that is already occupied by a file copied via `Containerfile` creates a conflict or a broken reference. | **Prefer** direct `COPY` in `Containerfile` for static config and use `tmpfiles.d` only for ephemeral/stateful paths (`/var`, `/run`). Avoid overlapping declarations. |
| **DMS-Greeter CLI Incompatibility** | Passing `--path` pointing to a config directory instead of the QML source directory causes an immediate crash of the greeter. | **Verify** CLI flags against official docs. Use `--cache-dir` for settings and omit `--path` unless using custom shell sources. |
| **Greeter Input Group Missing** | Compositores like Niri running as the `greetd` user cannot access devices without the `input` group. | **Always** add `greetd` to the `input` group in addition to `video` and `render` in the `Containerfile`. |
| **Audio Sink Sync (Deaf Slider)** | Decoupled volume between EasyEffects (software) and ALSA (hardware) results in non-functional sliders if hardware gain is low. | Enable **Volume Tracking** in EasyEffects global settings to synchronize software gain with the underlying hardware sink. |
| **Protected Package Swap Constraints (DNF5)** | DNF5 prevents the removal of critical packages (e.g., `sudo`) defined in `protected_packages` even if a replacement is provided in the same transaction. | Use `--setopt=protected_packages=` to temporarily clear the protection list during the swap transaction. Prefer a single transaction with `--allow-erasing`. |
| **Repository Asset Persistence** | Over-aggressive manual cleanup of build artifacts can lead to the accidental deletion of tracked repository assets (like `.der` certificates). | Distinguish between untracked build secrets (to be removed) and tracked public assets (to be preserved) before executing cleanup commands. |
| **Unchecked Pipe Execution (Curl to Shell)** | Piping `curl` output directly to `sh" without the `-f` flag causes the shell to execute error messages (e.g., "error: 404") if the download fails. | **Always** use `curl -fsSL` when piping to a shell. Explicitly install all download/extraction utilities (`tar`, `xz`, `jq`) in the `builder` stage. |
| **Kernel Install Dracut Failure** | DNF attempts to regenerate the initramfs via `dracut` during kernel installation, which fails in containers due to missing `modules.dep` or environment context. | Disable automatic generation by creating `/etc/kernel/install.conf` with `initrd_generator=none` before installation. Then, manually run `depmod` and `dracut` after the RPM transaction. |
| **Multiple Kernels in Bootc Lint** | `bootc container lint` fails if multiple kernel versions are found in `/usr/lib/modules/`. | Explicitly remove all existing kernel packages and wipe `/usr/lib/modules/*` before installing a new custom kernel. |
| **Disk-Specific I/O Scheduling** | Generic I/O scheduler rules may apply sub-optimal schedulers to rotating vs. non-rotating drives. | Use explicit udev rules checking `ATTR{queue/rotational}` to apply `bfq` to HDDs and `adios` (or similar) to SSDs/NVMe for optimal responsiveness. |
| Tarball Path Structure Mismatch | Failure to account for nested directories (e.g., `bin/` or versioned folders) within release tarballs leads to broken `COPY` instructions in multi-stage builds. | Always use `tar -tf` to audit the archive structure. Use `--strip-components` to flatten the directory hierarchy or adjust the `COPY` source paths to match the actual extraction results. |
| **Secret Exposure in Config Files** | Committing plain-text credentials (WiFi PSK, EAP identities) in NetworkManager `.nmconnection` files. | **Never** commit files containing actual secrets. Use `.gitignore` for local configurations and provide only sanitized templates if necessary. |
| **Polkit UX vs Security Conflict** | Enforcing `AUTH_ADMIN_KEEP` on frequent background operations (like USB mounting, app updates, or dynamic kernel scheduling) introduces excessive password prompts that break the frictionless desktop experience. | Use `Result.YES` for trusted local tools (e.g., udisks, flatpak, scx) where UX priority supersedes strict isolation boundaries. Ensure group restriction (e.g., `isInGroup("wheel")`) is maintained. |
| **Firewall Over-Exposure** | Enabling SSH on the `public` zone by default increases the attack surface for remote brute-force. | Keep `ssh` disabled in the default `public` zone. Users should enable it explicitly or use a more restrictive zone/VPN. |
| **Stale Direct Binaries** | Using hardcoded versions and checksums for external binaries (like `sudo-rs`) leads to security debt and manual maintenance overhead. | **Always** use the GitHub API to dynamically retrieve the `latest` release. Extract the `digest` (SHA256) directly from the API metadata to automate integrity verification without hardcoding hashes. |
| **User-Level Automation Consistency** | Global system services cannot access user-home resources (like `~/Downloads`) due to security boundaries. Placing user units in `/etc/skel/` prevents `systemctl --global enable` from finding them during build. | Deploy user-level systemd units to `/etc/systemd/user/` (not `/etc/skel/`) and enable them using `systemctl --global enable` in the build manifest. This ensures the service is found at build time and is active by default for all user sessions. |
| **Immutable Path Conflict in Distrobox** | Extracting archives to shared system paths (e.g. `/usr/local/share`) inside containers fails when the container engine mounts host system asset paths read-only (common in Atomic OS). | Extract to isolated, writable directories (e.g. `/opt/<tool>`) and only symlink the final executable to `/usr/local/bin`. |
| **Terminal Split Proliferation** | Calling terminal-spawning functions (like `open`) repeatedly in custom editor keymaps without verifying active session instances creates redundant split panes. | Use session retrieval APIs (like `get`) to identify existing active panes, and conditionally reveal or focus them instead of spawning new instances. |
| **Legacy Autocompletion Bridging** | Modern completion engines (like `blink.cmp`) cannot directly consume older source formats (like `nvim-cmp` sources) without compatibility adapters. | Bridge legacy plugins (e.g. `cmp-pandoc-references`) using a shim layer (e.g. `blink.compat`) and register them as custom providers within the modern engine. |
| **Ephemeral Background Services & Contextual Bindings** | Running auxiliary live-reloading services (like `quarto preview`) inside terminal windows splits the workspace unnecessarily. Triggering code-evaluation shortcuts without checking session state requires manual starting. | 1. Run live-reloading servers as headless background processes using API calls like `jobstart`. 2. Automatically launch connection backends on buffer loading (e.g. `auto_start = "always"`). 3. Use abstract syntax tree (AST) range queries (Treesitter) to limit REPL-sending keys (like `<Enter>`) to specific code-chunk blocks, invoking native plugin Lua APIs (e.g. `require("r.send").line()`) directly to bypass unstable simulated key mappings (like `<Plug>`), falling back to default navigation elsewhere. |
| **Unwanted Autopair Insertion** | Global auto-pair plugins (like `mini.pairs`) automatically close specific characters (like backticks) in all buffers, disrupting prose writing or code block editing in markup languages. | Override the character's insert mode mapping locally in buffer-local FileType autocommands (e.g., mapping `` ` `` to `` ` `` with `buffer = true`) to bypass the global autopair trigger. |
| **Bootc User/Group Conflicts** | Creating system users or groups in `/etc/passwd` at build time triggers 3-way merge conflicts and UID drift during bootc upgrades. | Segregate all build-time created users and groups (except root/wheel) to `/usr/lib/passwd` and `/usr/lib/group`, relying on the `altfiles` NSS backend for runtime resolution. |
| **Chunked-OCI Incremental Updates** | Standard flat images result in huge, monolithic layer downloads during upgrades. | Use `rpm-ostree compose build-chunked-oci --bootc --max-layers 127 --format-version 2` to split the OS image into granular, package-level OCI layers. |



## Package Verification (Recommended Workflow)
Before modifying the `Containerfile`, the agent should simulate or verify package names:
- Check if the package is available for the target Fedora version.
- Verify if it requires specific COPR repositories that are already present or need to be added.
| Errore | Causa | Soluzione |
| --- | --- | --- |
| Allineamento Monitor | Contatto su un solo vertice o snap orizzontale forzato | Garantire bordi condivisi e disattivare lo snap automatico di DMS per layout verticali |
| Keyring non sbloccato | Trattino '-' in PAM o mancanza di variabili D-Bus | Rimuovere '-' da pam_gnome_keyring e usare dbus-update-activation-environment |
| Rimozione pacchetti protetti (DNF5) | DNF5 blocca la rimozione di pacchetti critici (come `sudo`) | Usare `--setopt=protected_packages=` (globale) e `--allowerasing` (dopo il comando install) |
| Asset tracciati rimossi per errore | Pulizia manuale troppo aggressiva dei file di build | Distinguere tra segreti (da cancellare) e asset pubblici tracciati (da mantenere) |
| Esecuzione pipe non verificata (Curl) | Piping di `curl` in `sh` senza `-f` esegue messaggi di errore | Usare sempre `curl -fsSL` e installare esplicitamente `tar`, `xz`, `jq` nello stage builder |
| Fallimento Dracut durante installazione Kernel | DNF tenta di rigenerare l'initramfs in un container | Disabilitare la generazione automatica con `initrd_generator=none` in `/etc/kernel/install.conf` e poi lanciare `depmod` e `dracut` manualmente |
| Kernel Multipli in Bootc Lint | `bootc container lint` fallisce se trova più versioni del kernel in `/usr/lib/modules/` | Rimuovere esplicitamente i pacchetti kernel esistenti e pulire `/usr/lib/modules/*` prima di installare il nuovo kernel |
| Scheduling I/O differenziato | Regole udev generiche possono applicare scheduler non ottimali a dischi diversi | Usare regole udev che controllano `ATTR{queue/rotational}` per distinguere tra HDD (bfq) e SSD/NVMe (adios) |
| Disallineamento Percorsi Tarball | Mancata verifica della struttura interna degli archivi (es. sottocartelle `bin/`) porta al fallimento delle istruzioni `COPY`. | Usare sempre `tar -tf` per ispezionare l'archivio. Utilizzare `--strip-components` per appiattire la gerarchia o correggere i percorsi di origine nel `COPY`. |
| Nomi pacchetti non verificati (COPR) | Assumere nomi pacchetti (es. `-modules-extra`) senza verifica empirica | Verificare sempre i nomi esatti nei COPR, che spesso non seguono gli standard Fedora |
| Conflitto di Scrittura in Cartelle Condivise (Distrobox) | L'estrazione di tarball in cartelle come `/usr/local/share/...` all'interno del container fallisce se il gestore (Distrobox) le monta in sola lettura dall'host (comune su Fedora Atomic). | Estrarre i binari/asset in cartelle isolate e scrivibili del container (es. `/opt/`) e creare esclusivamente i link simbolici in `/usr/local/bin/`. |
| Moltiplicazione Split Terminale | Chiamare funzioni che creano terminali (come `open`) senza verificare se esiste già un'istanza attiva genera split ridondanti. | Usare API di recupero sessione (come `get`) per identificare l'istanza esistente e renderla visibile o focalizzarla condizionalmente. |
| Autocompletamento Citazioni (.bib) | I motori di completamento moderni (come `blink.cmp`) non supportano direttamente sorgenti legacy di `nvim-cmp`. | Utilizzare uno strato di compatibilità (come `blink.compat`) per mappare la sorgente legacy (`cmp-pandoc-references`) in `blink.cmp`. |
| Gestione Servizi in Background e Keymap Contestuali | Eseguire servizi ausiliari (come `quarto preview`) in terminali visibili riduce lo spazio dell'editor. Inviare comandi senza avviare la sessione richiede comandi manuali di setup (come `\rf`). | 1. Avviare i server in background come job headless tramite `jobstart`. 2. Abilitare l'avvio automatico della sessione interattiva al caricamento del buffer (`auto_start = "always"`). 3. Mappare Invio (`<CR>`) per l'invio del codice usando Treesitter per rilevare i blocchi di codice ed eseguendo direttamente le API Lua del plugin (evitando tasti virtuali `<Plug>` che potrebbero non risolversi tramite `nvim_feedkeys`), lasciando il comportamento standard altrove. |
| Auto-completamento Caratteri Indesiderato (Pairs) | I plugin di auto-pairs globali (come `mini.pairs`) inseriscono automaticamente la chiusura di caratteri come il backtick in tutti i buffer, interferendo con la scrittura di prose o blocchi di codice. | Sovrascrivere la mappatura in modalità inserimento localmente al buffer negli autocomandi `FileType` (es. mappando `` ` `` su `` ` `` con `buffer = true`) per bypassare l'autopair globale. |
| Allineamento Layout R & Math (Colonna Sinistra) | I tentativi di dividere direttamente finestre speciali come Snacks Explorer falliscono, causando l'apertura di colonne adiacenti e restringendo l'explorer a larghezza 5. | Lasciare che `R.nvim` apra l'Object Browser nella sua posizione originaria, quindi prenderne il buffer, chiuderne la finestra provvisoria ed inserirlo sotto Snacks Explorer tramite `nvim_win_call`. Fare lo stesso per la Math Preview. |
| Markdown Math Preview & Alterazione Caratteri | Attivare il rendering su tutto il file forza `conceallevel = 3` e sostituisce graffe `{}` e `<-` con caratteri Unicode speciali che si mostrano come quadrati rotti nel terminale (se mancanti nel font). La funzione `preview.condition` che restituisce `false` viene convertita in `nil` dal bug `to_static` di markview (`ok and res or nil`), bypassando il filtro. | NON usare `preview.condition` con valori `false` in markview.nvim. Rimuovere `quarto` da `preview.filetypes` e lasciare solo `markdown`. Il buffer preview usa `filetype=markdown`. Impostare `vim.opt.conceallevel = 0` globalmente in `options.lua`. |
| Buffer Wipeout su Window Close | Chiudere una finestra con buffer scratch (`buftype=nofile`) elimina il buffer se `bufhidden` non è impostato. Il codice che riusa il buffer dopo la chiusura ottiene `Invalid buffer id`. | Impostare `bufhidden=hide` sul buffer PRIMA di chiudere la finestra con `nvim_win_close`. Validare con `nvim_buf_is_valid()` dopo. |
| Race Condition Relocazione Finestre | R.nvim usa `vim.schedule` internamente per ridimensionare finestre. Chiamare `nvim_win_call` subito dopo `browser.start()` crea race condition. | Avvolgere la logica di relocazione in `vim.schedule(function() ... end)`. |
| Parser Treesitter Mancante | `markview.nvim` richiede il parser `latex` per renderizzare formule. `TSInstall latex` fallisce senza il CLI `tree-sitter`. | Installare `sudo npm install -g tree-sitter-cli` poi `TSInstall latex`. Verificare `~/.local/share/nvim/site/parser/latex.so`. |
| Conflitti Utenti/Gruppi Bootc | Creare utenti o gruppi di sistema in `/etc/passwd` durante la compilazione dell'immagine causa conflitti di merge ostree/bootc ad ogni upgrade. | Spostare a fine build tutti gli utenti/gruppi creati (tranne root/wheel) in `/usr/lib/passwd` e `/usr/lib/group` affidando la risoluzione a `altfiles` (nss-altfiles). |
| Aggiornamenti Incrementali OCI Lenti | L'uso di rechunker legacy genera invalidazioni casuali dei chunk aumentando la banda utilizzata per gli update. | Utilizzare il rechunker nativo `rpm-ostree compose build-chunked-oci --bootc --max-layers 127 --format-version 2` per suddividere l'immagine in layer OCI semantici per pacchetto. |
